## Codebase Patterns
- Use workflow job name `Lint and Test` as the required branch-protection status check for `main`.
- Use `npm run harness:high-risk -- --init --force` to reset `high-risk-harness.json`, then `npm run harness:high-risk` to produce scenario-level PASS/FAIL evidence.
- Normalize occlusion setting values through `scripts/occlusion-mode.js` before runtime use so invalid `cpuN` values safely fall back to `cpu6`.
- Use `npm run harness:occlusion-baseline -- --init --force` to regenerate `occlusion-baseline-run.json`, then `npm run harness:occlusion-baseline` to verify release baseline evidence completeness.
- On occlusion teardown, remove any stage children named `debug-*` to avoid stale debug overlays across scene transitions.
- Use optional catch binding (`catch {}`) for intentional no-op teardown guards to avoid extra unused-error lint warnings.
- Dynamic tile overlay policy for US-004: render overlays only when a token is actively controlled; keep layers empty when no token is controlled.
- For linked-wall dynamic tiles, refresh overlay on both `updateWall` (door/geometry/flag changes) and `deleteWall` events when the wall id is linked by any tile.
