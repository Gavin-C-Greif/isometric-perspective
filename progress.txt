## Codebase Patterns
- Global config lives in `scripts/consts.js` as `isometricModuleConfig` (MODULE_ID, DEBUG_PRINT, WORLD_ISO_FLAG, FOUNDRY_VERSION)
- Config values are initialized in the `Hooks.once("init")` handler in `scripts/main.js`, BEFORE register*() feature calls
- Version-conditional code uses `isometricModuleConfig.FOUNDRY_VERSION === 11` to branch between v11 and v13 APIs
- Foundry v11 vs v13 API differences: wall endpoints (wall.A/B vs wall.edge.a/b), visibility testing (canvas.effects.visibility vs canvas.visibility), token sorting API
- `module.json` compatibility is set to minimum v13 — v11 branches are legacy compat code
- No build step for JS — plain ES modules loaded directly by Foundry VTT
- All scripts are in `scripts/` directory, entry point is `scripts/main.js`
- Projection types and isometric constants are defined in `scripts/consts.js`
- Use `node --check <file>` for basic syntax validation (no tsconfig/typecheck infra yet)
